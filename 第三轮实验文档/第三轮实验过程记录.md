#### 2.2 improved prompt 分析

improved prompt包含角色定位、任务结构化定义（从五个维度定义口述史特征）、负面约束、示例引导、思维链分析、结构化输出。

在进行实验时，原以为improved prompt-v1会更优，因为v1思维链中的指令更加具体、明确、细致，然而实际上improved prompt-v2更优。

分析为：v1的prompt过度工程化：将思维过程外化为强制步骤，打断了自然的创作流程
而v2内化思维过程，将思维过程内化在模型内部，不强制外显，模型可以更自然地整合信息，而不是机械地按步骤输出

```
v1的问题：指导过于具体化
v1中过于具体的词汇指导可能限制创造性
"可以适当使用'那时候'、'那会儿'、'记得'等老人常用的口语词汇"
这种具体词汇列表可能让模型过度关注这些特定词汇的使用。

v2的优势：原则性指导
v2提供原则性指导："保持老人朴实自然的语言风格"，让模型根据上下文自然选择合适的表达方式。

为什么v2产生了更好的口述史？
更好的创作自由度
v2给了模型更大的创作空间，允许它基于对老人语言风格的理解，自然地选择合适的表达，而不是机械地插入特定词汇。
更符合人类创作习惯
口述史的整理本质上是一个创造性过程，v2的三步法（信息挖掘→结构规划→最终成文）更接近人类传记作家的实际工作流程。

这个结果符合认知负荷理论：
v1给模型施加了过多的过程性认知负荷
v2减少了外部认知负荷，让模型集中资源于核心创作任务

同时也符合创造力理论：
适度的约束（v2）比过度约束（v1）更能激发创造性输出
```

#### 3.basic prompt 与 improved prompt的区别

BASE_PROMPT_TEMPLATE 分析：
角色设定：“专业的口述历史整理者”。
方法：使用思维链步骤，分为五步：提取时间线、分析人物关系、梳理事件逻辑、理解情感基调、基于分析改写。
重点：强调保留细节、个人化表达、字数要求。
结构：直接给出步骤，然后要求改写，没有额外的任务定义或避免事项。
输出格式：没有指定输出格式，只是要求改写内容。

IMPROVED_PROMPT 分析：
角色设定：“富有经验的传记作家和历史学者”，给了模型更专业的身份，引导更深入的叙事处理。
任务定义：详细定义了口述史的五个要素：时间脉络、主题聚焦、情感与细节、叙事性、语言风格。这提供了全面的指导。
避免事项：明确列出了禁止虚构、禁止罗列、风格不当、遗漏重点，这有助于避免常见错误。
步骤：分为三个步骤：信息挖掘、叙事规划、最终成文。步骤更简洁，但结合了任务定义。
输出格式：指定了结构化的输出，包括信息梳理结果、文章大纲和口述史正文。这确保了输出的组织性。
语言风格强调：特别强调保持老人朴实自然的语言风格，使用口语词汇。

综上，basic prompt 与 improved prompt的区别：
IMPROVED_PROMPT角色设定：赋予模型更具体的角色（传记作家和历史学者），这鼓励模型以更专业、更叙事化的方式处理内容，而不是简单的整理。
任务定义更全面：通过明确口述史的核心要素，模型更好地理解什么是好的口述史，包括时间线、主题、情感、叙事性和语言风格。这引导模型生成更连贯、生动的文本。
避免事项减少错误：列出具体要避免的事项，如虚构、罗列、风格不当，帮助模型避免生成生硬或不准确的内容。
步骤更整合：IMPROVED_PROMPT的三个步骤（信息挖掘、叙事规划、最终成文）更符合创作流程，而BASE_PROMPT的五个步骤可能过于机械，导致输出碎片化。
语言风格强调：IMPROVED_PROMPT明确要求保持老人的语言风格，使用口语词汇，这直接导致了生成文本更自然、更贴近真实口述。
输出结构化：指定输出格式确保模型提供完整的信息梳理和大纲，这有助于模型在生成正文前有清晰的规划，从而产生更结构化的口述史。

1956年我大二时，复旦研制出中国第一台自主设计的计算机"复旦601"。这台机器主要是物理系老师研发的，我们学生只是听说这件事。当时计算机是很新奇的东西，大家都觉得能做出来是件大事，但对我个人影响不大，我还是继续学数学。

1958年，上海市在复旦办了一个计算机数学训练班。这个班比我们晚，应该是给五六级学生办的。我们55级是第一批五年制学生，前面54级是四年制，所以59年正好有个空档期。我就是在59年1月提前毕业留校的，因为大跃进的缘故，实际只读了三年多，算是大专学历。

那时候还没有计算机系，我就留在数学系搞概率统计。这个选择跟1956年国家制定的科学发展规划有关。当时中苏关系好，制定规划时有苏联专家参与。他们觉得中国的数学比较"陈旧"，国外一些重要数学分支我们这里要么没有，要么发展得不好。从国外回来的老专家也这么认为。所以规划里重点提出要加强三个方向：计算数学（跟计算机有关的数学）、微分方程和概率统计。这些领域在国外很受重视，但国内比较弱。

按照这个规划，数学系后来扩大了，国家也给了支持，概率统计教研组就是那时候成立的。当时系里从我们年级抽了8个学生提前毕业，分到需要加快发展的方向：三个搞计算机，三个搞运筹学，还有两个搞概率统计，我就是其中之一。

59年开始做概率统计后，六七十年代我参与了一个跟上海水文地质大队合作的项目。当时上海地面沉降很厉害，夏天经常积水。因为上海海拔本来就不高，一沉降就容易淹水。水文地质大队积累了大量资料，研究发现主要原因是夏季大量抽取地下水。地层分很多层，地下水主要在沙层里流动，抽水导致沙层压缩，层层累积就造成地面沉降。

他们想找出抽水量、灌水量与地下水位的关系，以及地下水位与沉降量的定量关系。我们做了大约两年，用最小二乘法做回归分析，这跟现在的人工智能原理很像。数据量很大，必须用计算机计算。我们尝试把问题简化，只分析沉降最严重的点，但想做整个面的时候就不行了，一方面是计算机硬件跟不上，另一方面我们的知识储备也不够。

当时复旦一直在研发计算机，从601到602，后来想搞753但没成功。计算机发展经历了电子管、半导体、小规模集成电路等阶段。75年成立计算机系时，把数学系和物理系相关老师都集中到了现在的逸夫楼。数学系老师主要搞计算数学、程序设计和软件应用，物理系老师主要搞硬件和自动控制。刚开始大家专业方向不同，融合需要过程。我那时研究方向也有些混乱，特别是文革期间还批判数学研究"脱离实际"。

这个口述记录完整呈现了受访者从求学、专业选择到参与重要科研项目的经历，既保留了个人的真实感受，也反映了特定历史时期的学科发展状况。通过具体事例展现了个人选择与时代背景的互动关系，语言朴实自然，细节丰富生动。

到了六十年代，我参与了一个很重要的项目——上海地面沉降研究。那时候上海地面沉降很厉害，夏天到处积水。上海本来就离海平面没多高，一沉降就麻烦了。水文地质大队研究多年发现，是因为夏天大量抽用地下水造成的。地层分好多层，地下水主要在沙层里流动，一抽水沙层就压缩，地面就沉降了。他们的办法是冬天往地下灌水，把地面撑起来。

我们当时要找出抽水量、灌水量和地面沉降之间的定量关系。这就要用到最小二乘法，其实跟现在的人工智能是一回事。我们要建立两个关系：一个是抽水量和地下水位的关系，另一个是地下水位和沉降量的关系。数据量特别大，上海布了很多测量点，每年定时测量，靠手算根本算不过来。这时候就必须用计算机了。

这个项目让我深刻体会到计算机的重要性。我们后来想从单点分析扩展到整个面的分析，但是做不下去，一方面是理论知识不够，另一方面计算机也跟不上。那时候复旦的计算机从601发展到602，后来想搞753但没成功。计算机技术发展很快，从电子管到半导体，再到小规模集成电路。电子管的机器体积很大，要占一个房间，很快就被淘汰了。

1975年复旦大学成立计算机科学系。为什么要成立？我觉得有几个原因：一是计算机在世界上已经是高新技术，中国相对落后；二是复旦在硬件、软件、应用方面都有相当基础了。硬件方面有719机，软件方面有自己的编译系统，应用方面像我们用地震方法找油的项目就很成功，当时还上了《人民日报》和《解放日报》。

成立计算机系的时候，把数学系和物理系的相关老师都集中到了现在的逸夫科技楼。这是王零书记推动的，他是学校领导，能拿出一栋楼来很不容易。刚开始，数学系和物理系的老师专业方向不同，数学系的有搞计算数学的、程序设计的、数据库的，物理系的有搞硬件的、自动控制的。要融合在一起需要个过程。

1981年到1984年我担任系主任，主要任务就是建立统一的教学框架。作为一个系，不能还是各搞各的，要有相对统一的基础课程。这段时间大家慢慢磨合，逐渐形成了计算机系的特色。

回想起来，我从一个数学系学生，到参与国家建设项目接触计算机，再到见证计算机系的成立和发展，最后自己也投身计算机视觉研究，这一路走来，真是跟着时代在进步。每个阶段都有不同的收获和体会，最重要的是始终保持着学习和探索的精神。
